Roast Battle App

A fun real time chat application made in Rails. Two users enter a timed “Battle” where each player uses wit and humor to insult their opponent. Other users watch the battle and vote for the funniest player. A timer is set at the beginning of a battle and a win-loss-knockout record is updated for each user at the end of the battle.

This was my final project for my coding bootcamp. The build time for this project was 2 weeks. The first week involved figuring out how to get around the non-asynchronous nature of rails. The first half of week two involved getting the flow of the game to work properly for each user persective (i.e. user who is logged in, user who is logged in and in a "battle", and user not logged in).

The second half of week two involved styling and deploying to heroku. Design was intended for a light-hearted, fun, and playful user experience. I thought the split screen design was good to represent a battle between two users. I also had a great time playing around with animations.

There's definitely much more that I can work on. As of now its 2 users in a battle and everyone else on the site. There needs to be a queue list for people wanting to join and maybe more than one "arena". I am not completely happy with the design and flow of the game so improvement can be made there. However, I made this from scratch in 2 weeks!! Given that, I am definitely proud of how it turned out.

Laced NY

A high end shoe app in Rails that scrapes from multiple name-brand sites using Nokogiri. I had so much fun making this app. Working with Nokogiri definitely got me involved in trying to read through a sea of code. Once we were able to find exactly what elements we needed to target, it was awesome to see the data being pulled onto our own page.

Another fun aspect of this site was to make the intro logo animation. Using SVG and path coordinates, I was able to create the cool fade in effect on page load. This is definitely something that I want to dive deeper into.

Each brand scraped is listed in a nav bar to the side. On click of each brand, the shoes will be filtered out accordingly. This was done using a sorting/filtering layout called isotope which is built on masonry.

This entire project was completed in one week with a group of three. This experience really improved my skills with Git and the whole process of working in a team setting.
